What Mobbin Is

Mobbin is the largest library of real, hand-picked mobile and web app screenshots — 300,000+ screens from top-tier iOS, Android, and web apps. It's an essential reference for UI designers: see exactly how Airbnb handles their checkout flow, how Figma does onboarding, how Notion structures their settings screen. Screenshots are organized by app, flow, and UI pattern. The platform itself (its own website and app) is also a masterclass in clean, functional dark UI.

Key Mobile Design Patterns (Distilled from Mobbin)

Onboarding Flows

  • 3–5 swipeable intro slides (progress dots below)
  • Value prop first, permissions request last
  • "Skip" always visible top-right
  • Social login (Apple, Google) before email — reduces friction
  • Progressive profiling: collect info post-signup, not during
  • Celebration screen after signup completion

Navigation Patterns

  • Tab bar: max 5 items, icons + labels, selected state with fill or tint
  • Bottom sheet navigation for secondary actions
  • Floating action button (FAB): primary action bottom-right
  • Swipe-to-go-back (iOS native behavior should not be blocked)
  • Header: title center, primary action right, back/close left
  • Context menus (long press) for secondary actions

Form Patterns

  • One question per screen for high-friction forms
  • Inline validation — show error immediately on blur, not on submit
  • Input groups: float labels animate up on focus
  • Show/hide password toggle always present on password fields
  • Smart defaults: auto-detect country, suggest email domains
  • CTA button disabled until form is valid

Empty States

  • Illustration (simple, not stock) + headline + subtext + CTA
  • Illustration size: 60–80% of screen width, centered
  • Tone: friendly and helpful, not apologetic
  • Specific CTA: "Add your first item" not "No items found"
  • Match app personality in illustration style
  • Search empty state: suggest alternatives or clear search
Mobbin Platform Design (Its Own UI)
  • Filter sidebar: Left sidebar with collapsible category sections — checkboxes with count badges (e.g. "Navigation (847)")
  • Screen grid: Tight 5–7 column grid of phone-frame screenshots with app icon and name below
  • App cards: App icon (rounded-xl), name below, screen count badge, "last updated" timestamp
  • Phone frame overlay: Subtle device frame (status bar, home indicator) added to every screenshot
  • Search: Instant search with dropdown showing app suggestions + pattern type suggestions
  • Keyboard shortcuts: Arrow keys to navigate screens, F to favorite, / to focus search
  • Preview modal: Click screen → full-page modal with scroll arrows, related screens below
What to Steal from Mobbin
  • Single-question forms: Break long forms into one-question-per-screen flows — conversion rates dramatically improve
  • Phone frame wrapper component: Display mobile screenshots in a device frame — adds professionalism to any mobile UI showcase
  • Filter sidebar with counts: Always show item counts next to filter options — helps users predict results before clicking
  • Disabled CTA until valid: CTA button starts grayed/disabled, enables once form validation passes — clearest UX signal
  • App icon + name card grid: Pattern for showing apps/tools in a reference grid — works for any catalog interface
  • Bottom sheet for actions: Instead of dropdown menus on mobile, use a bottom sheet with large touch targets — 44px minimum

Create a multi-step form component inspired by Mobbin's analysis of top onboarding flows. Each step shows one question fullscreen. Features: (1) progress bar at top showing step X of N, (2) question headline 28px weight-600, subtext 16px gray, single input or choice below, (3) "Continue" CTA button disabled until field is valid — transitions from gray to brand color when valid, (4) swipe left/right on mobile to navigate steps, (5) Framer Motion AnimatePresence with slide-in from right / slide-out to left transitions between steps, (6) final success screen with animation. Steps defined as an array of config objects: {id, question, type: 'text'|'choice'|'email', required}.

Phone Frame Screenshot Component

Create a PhoneFrame React component that wraps any content (or an image src) in a realistic iPhone-style device frame. The frame should be: rounded-[44px] bg-[#1a1a1a] border-2 border-[#2a2a2a] with an inner shadow. Include: top notch (pill shape, 40% width, height 20px, bg black, absolute top-0 center), status bar area (20px height showing mock time "9:41" and signal icons), home indicator bar at bottom (40% width, 4px height, bg rgba(255,255,255,0.3), rounded, margin auto). The content/screenshot fills the remaining inner area. Scale the frame with a width prop, aspect-ratio maintained at 9:19.5 (iPhone proportions).

Code Snippet — Mobbin-Style Filter Sidebar
// Mobbin-style filter sidebar with collapsible categories and counts
"use client";
import { useState } from "react";
import { motion, AnimatePresence } from "framer-motion";

interface FilterCategory {
  id: string;
  label: string;
  options: { id: string; label: string; count: number }[];
}

export function FilterSidebar({
  categories,
  selected,
  onToggle,
}: {
  categories: FilterCategory[];
  selected: Set<string>;
  onToggle: (id: string) => void;
}) {
  const [expanded, setExpanded] = useState<Set<string>>(
    new Set(categories.map((c) => c.id))
  );

  const toggle = (catId: string) => {
    setExpanded((prev) => {
      const next = new Set(prev);
      next.has(catId) ? next.delete(catId) : next.add(catId);
      return next;
    });
  };

  return (
    <aside className="w-60 bg-[#141414] border-r border-[rgba(255,255,255,0.06)] h-screen sticky top-0 overflow-y-auto p-4">
      {/* Active filters */}
      {selected.size > 0 && (
        <div className="mb-4">
          <p className="text-[11px] text-gray-500 uppercase tracking-widest mb-2">Active Filters</p>
          <div className="flex flex-wrap gap-1">
            {Array.from(selected).map((id) => (
              <button
                key={id}
                onClick={() => onToggle(id)}
                className="flex items-center gap-1 px-2 py-1 rounded bg-[rgba(245,158,11,0.1)] border border-[rgba(245,158,11,0.2)] text-[#f59e0b] text-[11px]"
              >
                {id} <span className="ml-1">×</span>
              </button>
            ))}
          </div>
        </div>
      )}

      {/* Categories */}
      {categories.map((cat) => (
        <div key={cat.id} className="mb-1">
          <button
            className="w-full flex items-center justify-between py-2 text-[13px] text-gray-400 hover:text-white transition-colors"
            onClick={() => toggle(cat.id)}
          >
            <span className="font-medium">{cat.label}</span>
            <span
              className="text-[10px] transition-transform duration-200"
              style={{ transform: expanded.has(cat.id) ? "rotate(180deg)" : "rotate(0deg)" }}
            >▾</span>
          </button>
          <AnimatePresence initial={false}>
            {expanded.has(cat.id) && (
              <motion.div
                initial={{ height: 0, opacity: 0 }}
                animate={{ height: "auto", opacity: 1 }}
                exit={{ height: 0, opacity: 0 }}
                transition={{ duration: 0.2, ease: "easeInOut" }}
                className="overflow-hidden"
              >
                {cat.options.map((opt) => (
                  <label
                    key={opt.id}
                    className="flex items-center justify-between py-1.5 px-2 rounded cursor-pointer hover:bg-[rgba(255,255,255,0.04)] transition-colors"
                  >
                    <div className="flex items-center gap-2">
                      <input
                        type="checkbox"
                        checked={selected.has(opt.id)}
                        onChange={() => onToggle(opt.id)}
                        className="w-3.5 h-3.5 accent-[#f59e0b]"
                      />
                      <span className="text-[13px] text-gray-300">{opt.label}</span>
                    </div>
                    <span className="text-[11px] text-gray-600">{opt.count.toLocaleString()}</span>
                  </label>
                ))}
              </motion.div>
            )}
          </AnimatePresence>
        </div>
      ))}
    </aside>
  );
}
