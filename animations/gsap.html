Animation Library

GSAP + ScrollTrigger

GreenSock Animation Platform — the industry-standard JavaScript animation library. ScrollTrigger is its most powerful plugin, enabling scroll-linked animations, pinning, parallax, and scrubbing. Used by Apple, Google, and Awwwards-winning sites.

Installation & Setup

GSAP core and ScrollTrigger are free. Club plugins (SplitText, MorphSVG, DrawSVG) require a paid membership.

bash
npm install gsap
# or: yarn add gsap  |  pnpm add gsap
javascript — register plugins (required)
import gsap from 'gsap';
import { ScrollTrigger } from 'gsap/ScrollTrigger';
import { ScrollToPlugin } from 'gsap/ScrollToPlugin';
import { TextPlugin } from 'gsap/TextPlugin';
import { Flip } from 'gsap/Flip';

// MUST call registerPlugin before using any plugin
gsap.registerPlugin(ScrollTrigger, ScrollToPlugin, TextPlugin, Flip);
html — CDN (no bundler)
<script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/gsap.min.js"></script>
<script src="https://cdnjs.cloudflare.com/ajax/libs/gsap/3.12.5/ScrollTrigger.min.js"></script>

Basic Tweens

The three tween methods are your daily drivers. GSAP transforms are GPU-accelerated and always use matrix3d under the hood.

javascript
// gsap.to() — animate current values TO the target
gsap.to('.box', {
  x: 200,           // translateX (px)
  y: -50,           // translateY (px)
  xPercent: -50,    // translateX (%)
  rotation: 360,    // rotate in degrees
  rotationX: 45,    // 3D rotateX
  scale: 1.5,       // scaleX and scaleY
  scaleX: 2,
  opacity: 0.5,
  width: '200px',   // CSS property
  backgroundColor: '#f59e0b',
  duration: 1,
  ease: 'power2.out',
  delay: 0.5,
  repeat: 2,        // repeat 2 more times (total: 3)
  yoyo: true,       // reverse on alternating repeats
  repeatDelay: 0.5, // pause between repeats
  onComplete: () => console.log('done'),
  onUpdate: () => {},
  onStart: () => {},
  onRepeat: () => {}
});

// gsap.from() — animate FROM these values to current CSS
gsap.from('.hero h1', { y: 60, opacity: 0, duration: 0.8, ease: 'power3.out' });

// gsap.fromTo() — explicit start AND end
gsap.fromTo('.card',
  { opacity: 0, y: 40 },
  { opacity: 1, y: 0, duration: 0.6, ease: 'power2.out' }
);

// gsap.set() — instant (no animation), good for initial state
gsap.set('.element', { opacity: 0, y: 20 });

// Kill a tween
const tween = gsap.to('.box', { x: 500, duration: 2 });
tween.kill();

// Targeting — all these work
gsap.to('.selector', { x: 100 });
gsap.to(document.querySelector('#el'), { x: 100 });
gsap.to(document.querySelectorAll('li'), { x: 100 });
gsap.to([el1, el2], { x: 100 });

Timelines & Position Parameter

Timelines sequence tweens. The position parameter is the most powerful feature — it controls exactly when each tween starts.

javascript
const tl = gsap.timeline({
  defaults: { ease: 'power2.out', duration: 0.6 },
  onComplete: () => console.log('sequence complete'),
  repeat: -1,    // -1 = infinite repeat
  yoyo: true,
  paused: true   // don't autoplay
});

// Position parameter — the key to orchestration
tl.from('.a', { x: -100 })           // appends to end (default)
  .from('.b', { x: 100 }, '<')       // "<" = same start as previous
  .from('.c', { y: 50  }, '<0.2')    // 0.2s AFTER previous STARTS
  .from('.d', { opacity:0 }, '+=0.5') // 0.5s AFTER previous ENDS
  .from('.e', { scale: 0 }, '-=0.3')  // 0.3s BEFORE previous ends
  .from('.f', { rotation:180 }, 1.5); // at exactly 1.5s mark

// Labels for readability
tl.addLabel('intro')
  .from('.logo', { opacity: 0 })
  .addLabel('content', '+=0.3')
  .from('.heading', { y: 30, opacity: 0 }, 'content')
  .from('.body', { y: 20, opacity: 0 }, 'content+=0.2');

// Timeline control methods
tl.play();
tl.pause();
tl.reverse();
tl.restart();
tl.seek(2);         // jump to 2 seconds
tl.progress(0.5);   // jump to 50%
tl.timeScale(2);    // play at 2x speed
tl.kill();          // destroy it

ScrollTrigger Basics

ScrollTrigger adds scroll-based triggering to any GSAP tween or timeline. The start/end strings use "[trigger edge] [viewport edge]" syntax.

javascript
import { ScrollTrigger } from 'gsap/ScrollTrigger';

// Attach to a tween
gsap.from('.section h2', {
  scrollTrigger: {
    trigger: '.section h2',    // element that triggers animation
    start: 'top 80%',          // [trigger edge] [viewport edge]
    end: 'bottom 20%',
    toggleActions: 'play none none reverse',
    // toggleActions: onEnter onLeave onEnterBack onLeaveBack
    // Possible values: play | pause | resume | reset | restart | complete | reverse | none
    markers: false,            // set true during dev to see trigger lines
    id: 'heading-trigger',
  },
  y: 50,
  opacity: 0,
  duration: 0.8,
});

// Start/end position strings:
// "top top"      — top of trigger element = top of viewport
// "top center"   — top of trigger element = center of viewport
// "top 80%"      — top of trigger at 80% down viewport
// "center center"
// "bottom top"   — bottom of trigger = top of viewport
// "top +=200"    — 200px below top of viewport
// "top -=100"    — 100px above top of viewport

// Standalone ScrollTrigger (callbacks only)
ScrollTrigger.create({
  trigger: '#section',
  start: 'top center',
  end: 'bottom center',
  onEnter: () => el.classList.add('active'),
  onLeave: () => el.classList.remove('active'),
  onEnterBack: () => el.classList.add('active'),
  onLeaveBack: () => el.classList.remove('active'),
  onUpdate: self => console.log('progress', self.progress.toFixed(2)),
  onToggle: self => console.log('active:', self.isActive),
});

// Refresh (call after dynamic content loads)
ScrollTrigger.refresh();

// Kill all (use in React cleanup)
ScrollTrigger.getAll().forEach(t => t.kill());

Scrub Animations

Scrub ties animation progress directly to scroll position — the animation plays forward/backward as you scroll. Essential for parallax and storytelling sites.

javascript
// scrub: true — instant link (no smoothing)
gsap.to('.parallax-img', {
  y: -200,
  ease: 'none',  // use 'none' for scrub
  scrollTrigger: {
    trigger: '.section',
    start: 'top bottom',
    end: 'bottom top',
    scrub: true
  }
});

// scrub: 0.5 — lag in seconds (smooth catch-up)
gsap.to('.text', {
  x: 300,
  scrollTrigger: {
    trigger: '.section',
    scrub: 0.5  // 0.5s to catch up to scroll position
  }
});

// Scrubbed timeline — the cinematic scroll pattern
const tl = gsap.timeline({
  scrollTrigger: {
    trigger: '#cinematic-scene',
    start: 'top top',
    end: '+=3000',   // 3000px scroll drives whole timeline
    pin: true,
    scrub: 1,
    snap: { snapTo: 'labels', duration: { min: 0.2, max: 0.5 } }
  }
});

tl.addLabel('start')
  .to('.circle', { scale: 5, opacity: 0 })
  .addLabel('reveal')
  .from('.headline', { opacity: 0, y: 40 }, '<')
  .to('.bg', { backgroundColor: '#000' })
  .addLabel('end')
  .to('.logo', { scale: 1.2, opacity: 1 });

Pinning & Horizontal Scroll

Pin an element in the viewport while the user scrolls through a defined range — essential for immersive sections.

javascript
// Simple pin
ScrollTrigger.create({
  trigger: '#hero',
  start: 'top top',
  end: '+=800',
  pin: true,
  pinSpacing: true  // adds margin below to prevent layout jump
});

// Horizontal scroll (most common pattern)
const panels = gsap.utils.toArray('.panel');
gsap.to(panels, {
  xPercent: -100 * (panels.length - 1),
  ease: 'none',
  scrollTrigger: {
    trigger: '.h-scroll-wrap',
    pin: true,
    scrub: 1,
    snap: 1 / (panels.length - 1),
    start: 'top top',
    end: () => '+=' + document.querySelector('.h-scroll-wrap').scrollWidth
  }
});

// Parallax layers using data attributes
gsap.utils.toArray('[data-parallax]').forEach(layer => {
  const speed = parseFloat(layer.dataset.parallax) || 0.5;
  gsap.to(layer, {
    y: () => -200 * speed,
    ease: 'none',
    scrollTrigger: {
      trigger: layer.closest('section'),
      start: 'top bottom',
      end: 'bottom top',
      scrub: true,
      invalidateOnRefresh: true  // recalculate on window resize
    }
  });
});

Stagger & Batch

Animate groups of elements with offset timing. Use ScrollTrigger.batch() for large lists to avoid creating hundreds of ScrollTriggers.

javascript
// Simple stagger
gsap.from('.card', {
  y: 40,
  opacity: 0,
  duration: 0.6,
  stagger: 0.1,  // 0.1s between each element
  ease: 'power2.out',
  scrollTrigger: {
    trigger: '.cards-container',
    start: 'top 75%'
  }
});

// Advanced stagger object
gsap.from('.grid-item', {
  scale: 0.85,
  opacity: 0,
  duration: 0.5,
  stagger: {
    amount: 0.8,       // total time distributed across all elements
    from: 'center',    // 'start' | 'center' | 'end' | 'random' | index
    grid: [3, 4],      // [rows, cols] for 2D grid animations
    axis: 'y'          // stagger along y axis in grid
  }
});

// Batch — best for large lists
ScrollTrigger.batch('.fade-up', {
  interval: 0.1,   // min time between batches
  batchMax: 5,     // max elements per batch
  onEnter: elements => {
    gsap.from(elements, {
      opacity: 0,
      y: 30,
      duration: 0.6,
      stagger: 0.08,
      ease: 'power2.out'
    });
  },
  start: 'top 90%',
  once: true
});

Eases Reference

Each ease has .in, .out, and .inOut variants. Some accept amplitude/period parameters.

EaseBest For
power1–4.outGeneral UI — the everyday default. Higher number = more dramatic
back.out(1.7)Elements popping in — slight overshoot then settle
elastic.out(1, 0.3)Springy/playful interactions, toggles
bounce.outBall-like physical bounce
circ.outFast entrance, very smooth decel
expo.outRapid start creeping to stop — dramatic hero reveals
sine.inOutSmooth organic motion, breathing/looping animations
none / linearScroll-scrubbed animations, progress bars, constant motion
steps(12)Frame-by-frame, sprite sheet animations
CustomEase.create('id', 'M0,0 C...')Brand-specific motion language, total bezier control
javascript — CustomEase
import { CustomEase } from 'gsap/CustomEase';
gsap.registerPlugin(CustomEase);

// Create once, reuse by name
CustomEase.create('brandEase', 'M0,0 C0.126,0.382 0.282,0.674 0.44,0.822 0.632,1.002 0.818,1.001 1,1');

gsap.to('.box', {
  x: 500,
  ease: 'brandEase',
  duration: 1
});

// Get ease path from https://gsap.com/docs/v3/Eases/CustomEase

React Patterns

Always use gsap.context() to scope animations to a container ref. Always return ctx.revert() from the useEffect cleanup.

jsx — correct React cleanup pattern
import { useEffect, useRef } from 'react';
import gsap from 'gsap';
import { ScrollTrigger } from 'gsap/ScrollTrigger';
gsap.registerPlugin(ScrollTrigger);

export function AnimatedSection() {
  const containerRef = useRef(null);

  useEffect(() => {
    // gsap.context() scopes all selectors to containerRef
    const ctx = gsap.context(() => {

      // Entrance animation
      gsap.from('.animate-title', {
        y: 50, opacity: 0, duration: 0.8, ease: 'power3.out',
        scrollTrigger: { trigger: '.animate-title', start: 'top 80%', once: true }
      });

      // Staggered cards
      gsap.from('.animate-card', {
        y: 30, opacity: 0, duration: 0.5, stagger: 0.1, ease: 'power2.out',
        scrollTrigger: { trigger: '.animate-card', start: 'top 85%', once: true }
      });

    }, containerRef); // scope = containerRef element

    return () => ctx.revert(); // cleans up all tweens and ScrollTriggers
  }, []);

  return (
    <section ref={containerRef}>
      <h2 className="animate-title">Title</h2>
      <div className="grid">
        <div className="animate-card">Card 1</div>
        <div className="animate-card">Card 2</div>
        <div className="animate-card">Card 3</div>
      </div>
    </section>
  );
}
jsx — counter animation hook
import { useEffect, useRef } from 'react';
import gsap from 'gsap';
import { ScrollTrigger } from 'gsap/ScrollTrigger';

export function AnimatedCounter({ target, suffix = '' }) {
  const ref = useRef(null);

  useEffect(() => {
    const obj = { val: 0 };
    const tween = gsap.to(obj, {
      val: target,
      duration: 2,
      ease: 'power1.out',
      onUpdate: () => {
        if (ref.current) {
          ref.current.textContent = Math.round(obj.val).toLocaleString() + suffix;
        }
      },
      scrollTrigger: {
        trigger: ref.current,
        start: 'top 85%',
        once: true
      }
    });
    return () => tween.kill();
  }, [target, suffix]);

  return <span ref={ref}>0{suffix}</span>;
}

GitHub README Notes

Key facts from the official GSAP GitHub repository (gsap/GSAP).

Performance: GSAP is approximately 20x faster than jQuery for animations. It uses requestAnimationFrame, hardware-accelerated transforms, and avoids layout thrashing by batching reads/writes.

Free plugins: ScrollTrigger, ScrollTo, Flip, MotionPath, Observer, Draggable, EasePack, CustomEase, TextPlugin, DrawSVG (basic)

Club (paid) plugins: SplitText, MorphSVG, DrawSVG (full), ScrollSmoother, GSDevTools, InertiaPlugin

ScrollSmoother: Official smooth scroll (Club only) — wraps content in a scrollable container. Alternative to Lenis for GSAP users on Club plan.

Version: GSAP 3.x (current as of 2026). GSAP 2.x is deprecated. Import from 'gsap' not 'gsap/TweenMax'.
javascript — useful utility methods
// gsap.utils — very useful helpers
gsap.utils.toArray('.items');          // NodeList to Array
gsap.utils.clamp(0, 100, value);       // clamp a value
gsap.utils.mapRange(0,1, 100,500, v); // map from one range to another
gsap.utils.wrap(0, 5, index);          // wrap index around range
gsap.utils.normalize(0, 100, value);  // normalize to 0-1
gsap.utils.random(10, 50, true);      // random number (true = snap to int)
gsap.utils.shuffle(array);            // shuffle an array in place
gsap.utils.interpolate(a, b, t);      // interpolate between values

// gsap.getProperty() — read current GSAP property
const x = gsap.getProperty('.box', 'x');
const opacity = gsap.getProperty('#el', 'opacity');

// gsap.quickTo() — ultra-fast repeated updates (mouse follow, etc.)
const moveX = gsap.quickTo('.cursor', 'x', { duration: 0.3, ease: 'power3' });
const moveY = gsap.quickTo('.cursor', 'y', { duration: 0.3, ease: 'power3' });
document.addEventListener('mousemove', e => { moveX(e.clientX); moveY(e.clientY); });
